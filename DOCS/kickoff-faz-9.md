## Purpose

Faz 9 kickoff requires an explicit decision for each of the six Sprint 1
narrative files. Per the Faz 9 atom map, every file must carry one of three
decisions: **reuse**, **refactor**, or **deprecate**. This document records the
decisions before any Faz 9 implementation atom merges.

## File-by-file decisions

### 1. `Assets/Scripts/Domain/Narrative/AskAboutTopic.cs`

- **Decision: refactor.**
- **Reason:** The current class binds `id + label + answer` directly. Faz 9
  Atom 2 promotes the id to `TopicId` (stable string value object) and Atom 3
  adds `TopicDef` (data row carrying prompt phrasing, gating predicate, default
  answer template id). The hard-coded `Answer` string disappears; answers are
  generated by `NpcDialogueService` from memory + faction + mood.
- **Action when Faz 9 starts:** delete `AskAboutTopic`, port its fixtures to
  `TopicDef` rows.

### 2. `Assets/Scripts/Simulation/Narrative/AskAboutService.cs`

- **Decision: refactor.**
- **Reason:** Current service performs a topic-id lookup against
  `SliceWorldState.Topics` and consults a single `Talker` actor. Faz 9 promotes
  this to `NpcDialogueService(asker, askee, TopicId)` which returns a typed
  `DialogueResponse(text, refusal_reason?)` pulling from `MemoryComponent`,
  faction relations, and current mood. The `Talker` slice field is already
  deprecated by Faz 1 stores.
- **Action when Faz 9 starts:** replace with `NpcDialogueService`, route
  through `ActorStore` + `MemoryComponent` + `FactionStore`.

### 3. `Assets/Scripts/Simulation/Narrative/AskDmService.cs`

- **Decision: deprecate.**
- **Reason:** Sprint 1 placeholder that returns hard-coded narrator strings.
  Faz 10 introduces the deterministic DM tool surface
  (`DmAgentToolSurface.query_world_snapshot`, `consult_fate`,
  `escalate_resolve_or_pass`) and Faz 12 adds the LLM proposal+validator
  pipeline. The Sprint 1 service has no migration target — its shape does not
  match the typed envelopes.
- **Action when Faz 9 starts:** mark the file with a deprecation comment;
  Faz 10 deletes it as part of the `dm-surface-and-escalation` bundle.

### 4. `Assets/Scripts/Simulation/Narrative/GuardInteractionService.cs`

- **Decision: refactor.**
- **Reason:** Guard-specific interaction (greet, intimidate, surrender)
  hard-codes guard-role behaviour. Faz 9 generalizes to topic-driven dialogue
  through `NpcDialogueService`; guard refusals become a memory + faction +
  mood result, not a separate service. Useful logic about crime threshold
  triggers carries forward into Faz 9 Atom 10 `MemoryWriteSystem` and Atom 11
  `TradeRefusalHook`.
- **Action when Faz 9 starts:** extract crime/threshold logic into the new
  `MemoryWriteSystem` + topic-gating predicates; delete the role-specific
  service.

### 5. `Assets/Scripts/Simulation/Narrative/NpcMemoryQueryService.cs`

- **Decision: refactor.**
- **Reason:** Existing query service has the right shape (recall facts by
  actor + topic) but binds to `NpcMemoryStore`/`InteractionEvent`/
  `TransactionRecord` shapes from Sprint 1. Faz 9 Atoms 4-7 introduce
  `MemoryFact`, `MemoryComponent`, and `MemoryRecallService` with stronger
  determinism (relevance threshold, deterministic ordering). The query
  contract carries forward; the storage shape changes.
- **Action when Faz 9 starts:** rename and reshape the service to
  `MemoryRecallService` consuming `MemoryComponent`; migrate fixtures.

### 6. `Assets/Scripts/Simulation/Narrative/ThinkService.cs`

- **Decision: deprecate.**
- **Reason:** Sprint 1 placeholder that emits a single internal-monologue
  string. Faz 9 does not implement actor inner monologue. Faz 12 LLM flavour
  layer may revisit, but the current shape has no migration target.
- **Action when Faz 9 starts:** mark for deletion in the same kickoff PR
  that introduces `MemoryRecallService`.
